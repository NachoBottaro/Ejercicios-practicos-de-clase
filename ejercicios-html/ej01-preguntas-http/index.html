Seccion preguntas sobre Http

  1. Sobre que protocolo binario está montado el protocolo HTTP?

    El protocolo http se encuentra montado sobre el protocolo Tcp.

  2. ¿Cuáles son los clientes http y los servidores http mas utilizados?

    Dentro de los clientes http mas usados nos encontramos con:

    • Los navegadores como mozila, Chrome, Safari, etc.
    • Curl

    Por otro lado, dentro de los servidores http, nos encontramos con los web servers como Nginx, Apache, CloudFlare, IIS, etc.

  3. ¿Que verbos admite un comando en el requerimiento HTTP?

    Algunos de los verbos que admite un comando de requerimiento http son:

    • GET
    • POST
    • DELETE
    • POST
  4. ¿Que contenido lleva el body de un requerimiento HTTP?

    El body de un requerimiento http puede contener datos de cualquier formato. Estos pueden ser datos ingresados en campos de un formulario o bien, files enviados en un proceso de upload

  5. ¿Que diferencia existe entre un URL y un URI?

    El URI identifica un recurso y lo diferencia de otros mediante un nombre, una ubicación o ambos. La URL identifica la dirección web o la ubicación de un recurso único. La URI esta dentro de la URL. La URI identifica un recurso, cualquier elemento que esta dentro de una pagina.

  6. ¿En que variables quedan almacenados datos de las respuestas http?

    En las cookies del navegador

  7. ¿Que significa Virtual hosting?

    El concepto de virtual hosting hace referencia a la virtualizacion de un servidor que tiene como objetivo almacenar varios dominios bajo la misma IP y cada uno tiene un nombre de host distinto.

  8. ¿Porque el HTTP se considera STATE LESS?

    Porque cada requerimiento y respuesta HTTP se realiza de forma independiente al resto de requerimientos y respuestas hechas previamente.

  9. ¿Como va a mejorar la velocidad de la WEB en poco tiempo?

    Con QUIC, ya que este protocolo realiza las funciones que hacía TCP y que no hace UDP. Además, lo hace de forma más eficiente, consiguiendo datos fiables, seguros y a mayor velocidad.

    QUIC, en un principio, se va a utilizar solo con el protocolo de aplicación HTTP/3, pero es probable que en el futuro otros protocolos de aplicación se apoyen en QUIC.

    HTTP/2 tambien, ya que este pide de una todo lo que necesita para cargar la pagina.

  10. ¿Como va a mejorar el comportamiento de una aplicación WEB en ambientes ruidosos o de débil conexión?

    QUIC las va a mejorar ya que este produce conexiones UDP mucho mas cortas, eficientes y de facil establecimiento, ideales para la transmisión de las partes constitutivas de un documento HTML.
